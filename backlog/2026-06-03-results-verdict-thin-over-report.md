# RESULTS.md retargeted — thin verdict over the auto-computed report

**Epic:** E5 — Observability & reporting
**Status:** IMPLEMENTED 2026-06-03 (RESULTS.md is the thin layer; FE-export realized by the published results page) — option (a) chosen; thin-verdict reframe to apply
when the first core-suite grid runs. FE exportable-report is a tracked follow-up.
**Date:** 2026-06-03
**Origin / triggered-by:** operator asked why we're aiming for a hand-maintained
RESULTS.md when Harbor already has reporting.

## Problem

The original "Definition of Done" (`2026-05-30-harness-vs-model-discriminating-suite.md`
§3) called for a **RESULTS.md** with per-category / per-difficulty / pass^k columns + a
verdict. But those numbers are **already produced automatically**:

- **Harbor's built-in reporting** — `harbor view`, per-job `result.json`
  (`metrics`, `reward_stats`, `pass_at_k`, cost/token), the `__openclaw`-vs-`__hermes`
  compare view, and the "Generate Analysis" button (Claude summary on subscription auth).
- **`metrics/track_a_weighted.py`** — the post-run analyzer that computes what Harbor's
  metric API can't (category weighting, the harness **split**, pass^k, efficiency,
  per-task discrimination) → writes `jobs/<job>/track_a_report.json`.

A hand-typed metrics table in RESULTS.md would therefore be **redundant** and a **drift
trap** — exactly the false-zero / cross-task-blending hazards just fixed in #93.

## Decision — option (a)

**RESULTS.md is demoted to a THIN verdict layer; the numbers stay where they're
computed.** It owns only what no automated report can:

1. **The verdict** — does the instrument separate harness from model, on which axes
   (reliability + efficiency, not single-run reward), and how big.
2. **Construct-validity caveats** — known asymmetries (honcho; **recall removed
   2026-06-03**), BLUNT controls, false-zero/VOID traps, the wipe-hook scope.
3. **Provenance + reproduction** — the one command that regenerates the numbers.

It **embeds the auto-computed split block from `track_a_report.json`** (pasted or
generated, never hand-curated) and **links to the viewer** for interactive exploration.
Source of truth for every number = Harbor reporting + `track_a_report.json`.

Rejected: (b) drop RESULTS.md entirely — loses the durable, git-tracked, human-authored
caveats + verdict that survive without the gitignored `jobs/` dir or a running viewer.

## Later — FE exportable report (follow-up, NOT this task)

Extend the front-end (viewer / dashboards) to **emit an exportable verdict file** (e.g. a
"Download report" that bundles the split + caveats + pass^k into a standalone artifact),
so the verdict isn't a manual paste. When that ships, RESULTS.md either becomes the
export target or is generated by it. Tracked as a new E5 todo.
